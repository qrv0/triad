# Triad Physics Implementation

This directory contains the reviewed physics implementation restored during the clean rebuild.

## Solver Lines

- `balanced_solver_3d.py` is the current rebuilt solver for the public anti-collapse/equilibrium simulation.
- `reference_solver_3d.py` preserves the historical Memory-NLS lineage from which the Triad solver emerged.

The balanced solver is the public entry point. The reference solver remains available so the development path is auditable.

## Numerical Structure

The balanced solver evolves a complex field with:

- kinetic propagation in Fourier space;
- attractive or repulsive instantaneous self-interaction;
- hierarchical memory fields;
- optional fractional spatial coupling;
- dissipation and fluctuation-dissipation noise.

The splitting order is:

```text
memory half-step
potential half-step
kinetic + dissipation full-step
potential half-step
memory half-step
FDT noise
```

This makes memory part of the evolution step instead of a post-step annotation.

## CUDA Notes

The solver uses NumPy by default and CuPy when `backend="cupy"` is selected. For local CUDA runs with pip-installed NVIDIA libraries, source:

```bash
source scripts/cuda-env.sh
```

before running the experiment scripts.

## Interpretation

The visual is not presented as a decorative animation. It is a controlled comparison:

- same initial field;
- same attractive nonlinearity;
- same numerical substrate;
- memory removed in one run and active in the other.

Without memory, the field forms localized collapse-like concentration. With hierarchical memory, collapse pressure releases into bounded distributed equilibrium.
