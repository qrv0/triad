# Archives

The `archives/` directory preserves the complete repository state before the clean rebuild.

## Current Snapshot

- `archives/pre-clean-2026-05-20_09-45-55/`

This snapshot contains the previous root structure, including documentation, experiments, outputs, interface drafts, methodology files, implementation code, and historical annotations.

## Why It Exists

The rebuild is not an erasure. The archive keeps the development ledger intact while allowing the public root to be reconstructed with a clearer role and cleaner reading path.

## How To Use It

When restoring material from the archive, classify it first:

- `core`
- `interface`
- `evidence`
- `implementation`
- `result`
- `archive-only`

Do not move a file back only because it existed before. Move it back because it serves the clean repository structure.
